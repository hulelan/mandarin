# Documentation — 读音练习 (Mandarin Pronunciation Practice)

Read **STATUS.md** first for current project state.

## Folder Structure

| Folder | Purpose | When to read |
|--------|---------|--------------|
| `STATUS.md` | Current state of all components, what works, what doesn't | First thing |
| `ROADMAP.md` | Vision, MVP phases, future features | "What are we building?" |
| `runbooks/` | Operational how-to guides (run locally, deploy, tunnel) | "How do I run this?" |
| `strategy/` | Design decisions, expansion plans, model evaluation | "Why did we do it this way?" |
| `references/` | Architecture deep-dives, research findings | "How does it work under the hood?" |
| `working/` | Active scratch space for in-progress research | Ephemeral — extract findings to other folders |
| `archive/` | Completed research, historical logs | "What did we try before?" |

## Lifecycle

Active research goes in `working/`. When conclusions are reached, extract them to `runbooks/`, `strategy/`, or `references/`, then move the original to `archive/`.
